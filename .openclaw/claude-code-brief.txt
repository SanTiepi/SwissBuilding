# Task: Post-Works Verification Tracker Implementation

## Context
You are implementing Q3.02 — a post-works verification system where contractors report completion of work items, capture photos with timestamps, and generate completion certificates. Read the full brief at `.openclaw/tasks/Q3-02-post-works-truth-tracker.md` in your workspace.

## Mission
Implement the complete post-works tracking pipeline: database models, validation service, API endpoints, mobile-first contractor form, and certificate PDF generation. All tests pass, dark mode included, one atomic commit.

## What this is / What this is NOT
- **IS:** end-to-end feature implementation spanning models→service→API→frontend pages
- **IS:** mobile-first contractor workflow (check items, upload photos, generate cert)
- **IS NOT:** a refactoring or cleanup task — this is new functionality

## Concrete Deliverables
1. **Backend models:** PostWorkItem + WorksCompletionCertificate + migration
2. **Backend service:** post_works_tracker_service.py with validation + completion % logic
3. **Backend service:** completion_certificate_generator.py using Gotenberg PDF
4. **Backend API:** POST /buildings/{id}/post-work-items/{id}/complete, GET endpoints
5. **Frontend page:** PostWorksTracker.tsx (mobile-responsive, touch-friendly)
6. **Frontend component:** PostWorkItemCard.tsx (before photo, checkbox, upload, notes)
7. **Frontend integration:** Building Home shows completion widget
8. **Tests:** 12+ backend unit tests + 8+ frontend component tests
9. **All existing tests must pass** (no regressions)

## Hard Rules
1. Copy existing patterns: models from `backend/app/models/diagnostic.py`, PDF from `backend/app/services/authority_report_generator.py`, photo upload from `frontend/src/pages/Contractor/MaterialPhotoCapture.tsx`
2. Mobile-first: touch-friendly inputs, large buttons, responsive layout (Tailwind + dark: classes)
3. Dark mode required: use `dark:` Tailwind classes throughout
4. Completion % = verified_items / total_items (show as progress bar in UI + endpoint)
5. Certificate PDF only appears at 100% completion
6. No type errors, no linting warnings after `npm run validate` and `ruff check/format`
7. Use existing auth/session patterns — contractor_id from JWT token
8. Before/after photo pairing stored in JSON: `[{before_id, after_id}]`
9. Verification score ≥80% confidence to auto-verify (else flag for manual review)
10. All new models must be added to `backend/app/models/__init__.py` exports
