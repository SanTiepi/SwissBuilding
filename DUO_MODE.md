# Duo Mode — Archived

## Status

As of `2026-03-29`, Duo Mode is no longer the active operating system for this repo.

Reason:
- the cleanup pass reached diminishing returns on a mature codebase
- the auto-generated backlog started drifting toward busy-work
- future work should be driven by business milestones, pilot blockers, or concrete bugs

## What Remains Useful

- `docs/duo_inventory.md` is kept as a historical inventory snapshot
- `DUO_TASKS.md` is kept as an archived record of the completed duo pass
- the branch `duo/improvements` is historical delivery work, not a standing workflow

## Current Rule

Do not resume autonomous continuation from `DUO_TASKS.md`.

Instead, prefer this priority order:
1. business milestone or pilot blocker
2. concrete product or extraction bug
3. user-directed or Codex-directed roadmap brief

Cleanup-only work should happen only when it directly reduces delivery risk, removes a blocker, or is explicitly requested.
