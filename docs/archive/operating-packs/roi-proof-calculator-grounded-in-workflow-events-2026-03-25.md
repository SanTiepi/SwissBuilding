# ROI Proof Calculator Grounded in Workflow Events

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding should be able to prove ROI from workflow events, not just from
generic promises.

## Rule

ROI should be grounded in:

- fewer dossier rebuilds
- earlier blocker discovery
- cleaner procedure loops
- less proof chasing
- more reuse

## Input classes

- time saved
- avoided repeated packaging
- avoided late-stage surprises
- reduced ambiguity
- reduced re-sending or re-explaining

## Acceptance

This pack is succeeding when ROI can be explained from real product events
rather than estimated vanity assumptions alone.
