# UX Information Hierarchy and Surface Grammar

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding wins only if information becomes easier to read, understand, and
act on.

This pack defines a shared surface grammar so product depth does not turn into
visual chaos.

## Rule

Every major surface should answer in this order:

1. what matters now
2. why it matters
3. what is proven
4. what can be done next

## Surface layers

### Layer 1 - immediate status

Should show:

- blocker
- deadline
- confidence
- freshness
- next action

### Layer 2 - explanation

Should show:

- why the status exists
- which rule, proof, or event caused it
- what audience or procedure it affects

### Layer 3 - drilldown

Should show:

- linked proof
- versions
- detailed history
- route or review path

## Visual priorities

- blockers beat background detail
- deadlines beat metadata
- confidence and freshness should be visible before deep drilldown
- evidence should be one step away, not buried

## Product anti-pattern

Do not design surfaces where:

- the user sees many cards but not the next action
- the proof exists but the reason is hidden
- the reason exists but the proof is hard to find

## Acceptance

The grammar is succeeding when a user can scan a page and answer:

- what matters now
- why
- what proves it
- what to do next
