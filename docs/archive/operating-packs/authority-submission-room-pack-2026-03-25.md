# Authority Submission Room Pack

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding should eventually make authority-facing work feel like a bounded
room with:

- current submission state
- current proof set
- request or response history
- explicit next move

## Rule

The submission room is not a second DMS.

It is the procedural and proof-focused room around one active authority path.

## Core elements

- submission summary
- current authority route
- active step
- required proof set
- sent and acknowledged items
- complement loop
- confidence and review notes

## Acceptance

This pack is succeeding when authority-facing work can be understood and moved
forward from one bounded procedural room.
