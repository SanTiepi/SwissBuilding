# Post-Sale Adoption Loop Pack

Date de controle: `25 mars 2026`

## Purpose

Selling SwissBuilding is not enough.

The product needs post-sale loops that turn an initial win into durable usage,
spread, and dependence.

## Loops

### Loop 1 - one workflow becomes daily habit

Examples:

- ControlTower
- procedure review
- proof pack generation

### Loop 2 - one actor invites another

Examples:

- manager invites contractor
- owner shares pack with fiduciary
- diagnostician publication creates downstream reuse

### Loop 3 - one successful proof reuse creates trust

Examples:

- same evidence supports authority and transaction pack

### Loop 4 - one resolved blocker proves product value

Examples:

- issue found early
- missing proof exposed in time
- authority request handled cleanly

## Acceptance

This pack is succeeding when SwissBuilding grows after sale because it keeps
proving itself in lived workflow, not because of account pressure alone.
