# Pilot Design Cookbook

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding should not enter pilots with vague success criteria.

This cookbook defines how to shape pilots that create real product learning and
real customer value.

## Rule

A pilot should be:

- narrow enough to finish
- deep enough to matter
- measurable enough to learn from

## Good pilot structure

- one persona lead
- one recurring workflow
- one clear blocker family
- one visible proof win
- one acceptance metric

## Bad pilot structure

- too many personas
- generic "let us try the platform"
- no recurring use
- no before or after comparison

## Acceptance

This cookbook is succeeding when pilots become evidence-producing, not just
relationship-maintaining.
