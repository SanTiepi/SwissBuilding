# Wow Moments and Category Differentiation Map

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding needs memorable moments where users immediately feel that it is
not just another building tool.

This map defines the strongest wow moments that are still grounded in real
product value.

## Rule

A wow moment is valid only if it creates lasting product value, not just demo
surprise.

## Wow moment 1 - One building, immediate clarity

User sees:

- blockers
- deadlines
- proof
- history
- next action

Why it differentiates:

- most tools still require context reconstruction

## Wow moment 2 - One proof, many audiences

User sees:

- one validated proof asset
- multiple safe reuse paths
- delivery and caveat trace

Why it differentiates:

- most tools still treat proof as a one-time file

## Wow moment 3 - Procedure becomes visible

User sees:

- active step
- missing proof
- complement loop
- route to authority

Why it differentiates:

- many tools stop at documents, not procedures

## Wow moment 4 - Ambiguity is explicit, not hidden

User sees:

- confidence
- manual review
- exact reason

Why it differentiates:

- most tools hide uncertainty until late failure

## Wow moment 5 - Batiscan report becomes durable building truth

User sees:

- publication lands in the building
- later packs and workflows reuse it
- operational truth changes

Why it differentiates:

- most reports still die as PDFs after delivery

## Acceptance

This map is succeeding when the strongest product moments are both memorable
and operationally real.
