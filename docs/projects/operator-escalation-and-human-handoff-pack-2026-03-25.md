# Operator Escalation and Human Handoff Pack

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding should know when to escalate to a human and how to make that
handoff clean.

This applies to:

- review-required rules
- procedural ambiguity
- identity ambiguity
- cross-product matching issues
- high-stakes proof questions

## Rule

Escalation should not feel like failure.

It should feel like controlled handoff with preserved context.

## Escalation contract

Every escalation should preserve:

- what triggered it
- what is known
- what is ambiguous
- what decision is needed
- what downstream step is waiting

## Typical escalation cases

- communal or cantonal ambiguity
- building match ambiguity
- stale proof in a high-stakes path
- authority request that needs expert response
- cross-product publication mismatch

## Product rule

Do not escalate with only a vague error message.

Escalation should carry the exact context a reviewer needs.

## Acceptance

This pack is succeeding when a reviewer can take over without reconstructing
the case from scratch.
