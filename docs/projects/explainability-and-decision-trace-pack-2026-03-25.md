# Explainability and Decision Trace Pack

Date de controle: `25 mars 2026`

## Purpose

SwissBuilding should not just show outputs.

It should show why an output exists.

This matters especially for:

- rules
- procedures
- proof reuse
- partner trust
- agent recommendations

## Rule

Any consequential output should have a decision trace.

## Minimum decision trace

- source or trigger
- projection or reasoning path
- confidence state
- affected object
- resulting next action or status

## Consequential outputs

- blocker
- obligation
- authority route
- review_required
- trust signal
- proof reuse opportunity
- benchmark signal

## Product rule

Do not require users to infer system reasoning from scattered screens.

The trace should be visible or one click away.

## Acceptance

The pack is succeeding when a user can ask:

- why is this blocked
- why is this due
- why is this pack missing something
- why is review required

and the product can answer without guesswork.
