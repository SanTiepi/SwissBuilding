# Pilot Scorecard and Exit Criteria Pack

Date de controle: `25 mars 2026`

## Purpose

Pilots should not end with a vague feeling.

They should end with a scorecard and explicit exit criteria.

## Rule

Every pilot should define:

- what success means
- what partial success means
- what failure means

## Scorecard dimensions

- recurring usage
- blocker clarity
- procedure clarity
- proof reuse
- actor spread
- trust gained

## Exit states

### Promote

- clear value proven
- repeatable workflow exists

### Extend narrowly

- value is visible
- but one blocker family still prevents generalization

### Stop

- no recurring habit
- no meaningful workflow lift

## Acceptance

This pack is succeeding when pilots end with a decision, not only a summary.
